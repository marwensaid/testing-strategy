Testing Strategy

Objectif du repository

Ce repository accompagne un cours de niveau Master 1 consacré à la stratégie de test logiciel et au développement dirigé par les tests (TDD).

Il a pour objectif de fournir un support pédagogique complet permettant de comprendre, expérimenter et maîtriser les différentes approches de test utilisées en environnement professionnel.

⸻

Ce que vous allez apprendre

Au fil des séances et des travaux pratiques, vous apprendrez à :

	•	Comprendre les enjeux du test logiciel (qualité, coût des bugs, gestion des risques)
	•	Concevoir une stratégie de test adaptée à un système
	•	Différencier et utiliser les principaux types de tests (unitaires, intégration, end-to-end, non-régression)
	•	Écrire des tests efficaces avec Java et Spring Boot
	•	Appliquer le TDD (Test Driven Development)
	•	Tester un système complet, incluant ses dépendances et ses contraintes réelles
	•	Aborder des sujets avancés comme la résilience, les crash tests et le chaos engineering

⸻

Contenu du repository

Le repository est structuré de manière progressive afin d’accompagner la montée en compétence :

Cours

Supports théoriques couvrant les concepts fondamentaux :

	•	stratégie de test
	•	types de tests
	•	bonnes pratiques
	•	qualité logicielle

Travaux pratiques (TP / TD)

Exercices guidés pour mettre en application les concepts :

	•	tests unitaires avec JUnit
	•	mocking avec Mockito
	•	tests Spring Boot
	•	tests d’intégration et d’API
	•	scénarios end-to-end

TDD

Exercices dédiés au développement dirigé par les tests :

	•	katas (FizzBuzz, Bowling, etc.)
	•	implémentation progressive de fonctionnalités

Cas d’étude

Mises en situation réalistes :

	•	conception d’une stratégie de test (par exemple un distributeur de billets)
	•	analyse de systèmes complets
	•	gestion des cas d’erreur et des situations extrêmes

Sujets avancés

Introduction à des pratiques industrielles :

	•	tests de charge
	•	crash tests
	•	chaos engineering

⸻

Organisation pédagogique

Le cours alterne entre apports théoriques, travaux pratiques, activités de groupe et études de cas. L’objectif est de développer à la fois des compétences techniques et une capacité de réflexion sur la qualité logicielle.

⸻

Stack technique

	•	Java
	•	Spring Boot
	•	JUnit 5
	•	Mockito
	•	H2 Database

⸻

Utilisation

Ce repository est conçu pour être utilisé comme support de cours, comme base de travail pour les étudiants et comme environnement d’expérimentation. Chaque dossier correspond à une étape du cours et contient les consignes, le code et, lorsque nécessaire, des éléments de correction.

⸻

Philosophie

Tester ne consiste pas uniquement à vérifier que le code fonctionne aujourd’hui, mais à s’assurer qu’il continuera de fonctionner demain.
